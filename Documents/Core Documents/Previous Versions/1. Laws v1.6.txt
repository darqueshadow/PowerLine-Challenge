0. Definitions (Immutable Glossary) 

The following terms establish mandatory, unambiguous definitions governing all PowerLine game construction. These definitions are legally binding within the scope of the PowerLine Challenge system. 
Claude must not reinterpret, alter, or override any definition in Section 0 unless Andrew explicitly instructs a temporary override. 
If any instruction appears to conflict with a definition in this section, Claude must pause and request clarification before proceeding. 

 

0.1 Empty Cartridge 

The neutral, unthemed base version of the game containing all core gameplay logic and placeholder visuals. The Empty Cartridge must never be modified directly. Claude must always duplicate the Empty Cartridge before creating or modifying any Cartridge. 

 

0.2 Cartridge 

A themed version of the game created from a duplicated Empty Cartridge. Cartridges may modify appearance, audio, and theme-specific effects but must not modify or replace core gameplay logic. 

 

0.3 Theme 

A cosmetic layer defining the visual and audio style of a Cartridge. Themes must never alter rules, scoring, dataset logic, challenge generation, difficulty progression, impact logic, or any other core system. 

 

0.4 Target 

A game object representing an actionable item in the playfield. A Target must always contain an attached Challenge. Targets must follow the core movement and lifecycle rules defined in the engine. 

 

0.5 Challenge 

A plain-language instruction attached to a Target. Challenge generation must follow the rules defined in the Data Sheet and Overview unless explicitly overridden by Andrew. 

 

0.6 Command 

The exact PowerLine syntax required to clear a Target. Commands must comply with CAD rules and must reflect the Command Template associated with the Challenge Template. 

 

0.7 Clear (Hit) 

The state in which a Target is correctly removed from play due to a valid Command input. 

 

0.8 Miss 

The state in which a submitted Command is invalid. The Target remains active unless cleared or impacted. 

 

0.9 Impact 

The state in which a Target reaches the lower boundary without being cleared. Impact behaviour must follow engine rules and must not be altered by themes. 

 

0.10 Data Sheet 

The CSV dataset containing command types, units, bases, templates, and data pools. Claude must parse and apply Data Sheet content exactly as defined, unless Andrew explicitly authorizes changes. 

 

0.11 Challenge Set 

A defined group of Challenge Types used for structured training. Challenge Sets must be loaded according to user selection and must not alter core logic. 

 

0.12 Challenge Template 

A plain-language template used to construct a Challenge. Must follow the structure defined in the Data Sheet. 

 

0.13 Command Template 

The template governing the formation of a required Command. Must align with the corresponding Challenge Template. 

 

0.14 Data Pool 

A set of values (Units, Bases, Hospitals, Call Numbers) used to populate templates. Values must be drawn exactly as defined. 

 

0.15 Developer Mode 

A restricted-access mode enabling internal testing. Developer Mode is activated via Ctrl + Shift + B and must remain hidden from regular users in non-demo builds. 

 

0.16 Blank Dataset Module 

A test-only dataset used solely in Developer Mode. Must remain inaccessible during normal gameplay. 

 

0.17 Glow Effect 

A contour-following outline applied to themed Targets. Glow must adhere to the silhouette of the PNG, rotate with the object, and must never default to bounding-box behaviour. 

 

0.18 Theme Injection Points 

The exclusive locations Claude is permitted to modify when applying a theme. These include Target appearance, Challenge appearance, Command UI styling, backgrounds, and optional cosmetic effects. No other engine components may be changed unless Andrew explicitly authorizes a temporary override.

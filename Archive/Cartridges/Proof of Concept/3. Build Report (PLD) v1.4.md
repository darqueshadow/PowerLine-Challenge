### 6. Impact & Projectile Logic - IMPLEMENTED ✓

**Dual Naming Convention:**
- `fireTowerProjectile()` = Tower initiates impact → **Tower Impact**
- `fireTargetProjectile()` = Target initiates impact → **Target Impact**
- `createGroundImpact()` = Target hits Zone → **Impact Zone** (Zone receives)

**Event Impact:**
- `createExplosion(x, y)` - occurs at actual collision point
- `createGroundImpact()` - occurs at Zone intersection
- Never at spawn points ✓

**Projectile Logic:**
- Origin calculated from firing object's position
- Destination calculated from target object's position
- Trajectory uses `Math.atan2()` for angle calculation
- Travel time based on distance
- Event Impact triggers on arrival

## 🎮 GAMEPLAY FLOW

### 1. Target Spawns
- Target (green rectangle) appears at top
- Challenge label (yellow rectangle) appears **empty** (no text for Blank dataset)
- Falls toward Zone (brown line at bottom)
- Animation duration = `fallDuration` (starts 15s, scales down)

### 2. Player Response
**Correct Command (empty string for Blank dataset):**
1. Player presses Enter with **empty input**
2. Tower fires projectile at Target
3. Projectile travels to Target
4. Explosion at collision point
5. Target removed
6. Points awarded
7. Streak continues

**Incorrect Command (any text for Blank dataset):**
1. Player types **anything** then presses Enter
2. Target fires projectile at Tower
3. Projectile travels to Tower
4. Tower receives impact
5. Shield decremented
6. Streak resets

**Missed Target (reaches Zone):**
1. Target opacity set to 0
2. Ground impact effect (mushroom cloud style)
3. Screen shake
4. Shield decremented
5. Target removed

### 3. Difficulty Scaling
- Every 1 correct: speed increases 7%
- Every 10 correct: add 1 more simultaneous target (max 6)
- Spawn rate: 1500ms → minimum 400ms
- Fall duration: 15000ms → minimum 8000ms

### 4. Game Over
- Occurs when shields reach 0
- Shows final score and accuracy
- Options: Play Again (1) or Main Menu (ESC)

## 🔧 KEY FUNCTIONS

### Spawn System
- `spawnTarget()` - Creates new Target with Challenge
- Collision avoidance (170px minimum horizontal spacing)
- Random position within play area bounds
- Prevents duplicate commands on screen

### Collision System
- `checkTargetPositions()` - Runs every 200ms
- Checks if Target reached Zone (time-based)
- Triggers Impact Zone event
- Decrements shields
- Removes destroyed targets

### Projectile System
- `fireTowerProjectile(target)` - Tower shoots at Target
- `fireTargetProjectile(target)` - Target shoots at Tower
- Calculates trajectory using trigonometry
- Animates travel with CSS transitions
- Triggers Event Impact on arrival

### Impact System
- `createExplosion(x, y)` - Radial explosion at point
- `createGroundImpact(target)` - Mushroom cloud + screen shake
- Always at collision point, never at spawn ✓

## 📝 DATASET

Game uses **Blank dataset** (for testing mechanics):

```javascript
const gameData = [
    { challenge: "", command: "", primaryTarget: "", additionalDetail: "" }
];
```

**Structure:**
- `challenge` = Empty string (challenge bubble displays but contains no text)
- `command` = Empty string (player presses Enter with no input to match)
- `primaryTarget` = Empty string (reserved for future use)
- `additionalDetail` = Empty string (reserved for future use)

**Blank Dataset Behavior:**
- ✅ **Correct answer**: Press Enter with empty input field
- ❌ **Wrong answer**: Type anything, then press Enter
- ❌ **Missed target**: Let Target fall to Zone without responding

This allows testing all three game outcomes without needing actual command data.
